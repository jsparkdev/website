description: 의존성 배열을 사용하지 않는 useEffect 훅과 이를 사용하지 않는 일반 작업의 차이에 대해 알아봅니다.
---

둘다 컴포넌트 렌더링, 리렌더링시 작업을 수행한다는 점은 같음.
단 타이밍이 좀 다름

- 유즈이팩트 : 렌더링 후에 실행됨, 비동기
  - 비동기라는게 브라우저가 컴포넌트를 다 그리고 마지막에 실행된다는 의미
  - 그래서 렌더링 출력에 영향을 주지 않는 데이터 페칭햐는 경우 등 초기 렌더링을 차단하지 않는 작업에 사용.
  - 즉, 렌더링 출력₩과 관련없는 작업에 사용.
- 일반작업 : 매 렌더링 시 실행됨, 렌더링과 동기적
  - 코드를 작성한 순서대로 실행되므로 요소를 그리기전에 실행됨.
  - 즉, 렌더링프로세스의 일부로 순서대로 실행됨.
  - 그래서 오래걸리는 작업의 경우 컴포넌트가 느리게 그려질 수 있음.
