# Cloudflare D1 + Drizzle ORM

Cloudflare D1 데이터베이스와 Drizzle ORM을 사용하여 데이터베이스를 생성하고 설정하는 방법에 대해 알아봅니다.

## Cloudflare D1 생성 및 설정

1. `wrangler` CLI를 사용하여 D1을 생성합니다.

```bash
pnpm wrangler d1 create <DB_NAME>
```

성공적으로 생성했다면, 아래와 같은 메시지가 콘솔에 출력됩니다.

```toml
[[d1_databases]]
binding = "DB" # i.e. available in your Worker on env.DB
database_name = "<DB_NAME>"
database_id = "<DB_ID>"
```

이 메시지를 `wrangler.toml` 파일에 붙여넣습니다. 이 파일은 `Workers`가 D1에 액세스하기 위한 정보를 제공합니다.

2. 프로젝트에서 D1의 타입을 생성하기 위해 아래 명령을 실행합니다.

```bash
pnpm wrangler types
```

## 스키마 생성 및 설정

`drizzle-orm` 및 `drizzle-kit` 패키지를 설치합니다.

```bash
pnpm add drizzle-orm
pnpm add -D drizzle-kit
```

`drizzle-orm`을 사용해 스키마 파일을 생성합니다.

```ts
import { relations } from "drizzle-orm";
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";

export const users = sqliteTable("users", {
  id: integer("id").primaryKey(),
  name: text("name"),
  email: text("email").unique().notNull(),
  username: text("username").unique().notNull(),
});

export const items = sqliteTable("items", {
  id: integer("id").primaryKey(),
  name: text("name").notNull(),
  description: text("description"),
  price: integer("price").notNull(),
  userId: integer("userId")
    .notNull()
    .references(() => users.id),
});

export const itemsRelations = relations(items, ({ one }) => ({
  user: one(users, {
    fields: [items.userId],
    references: [users.id],
  }),
}));
```

프로젝트 루트에 `drizzle.config.ts` 파일을 생성합니다.

```ts
import type { Config } from "drizzle-kit";

export default {
  schema: "./app/drizzle/schema.server.ts", // 스키마 파일 경로
  out: "./app/drizzle", // 마이그레이션 경로
  driver: "d1",
  dbCredentials: {
    wranglerConfigPath: "wrangler.toml",
    dbName: "<DB_NAME>", // D1 데이터베이스 이름
  },
} satisfies Config;
```

## 마이그레이션 생성 및 적용

작성한 스키마 파일을 기반으로 마이그레이션을 생성하기 위해 다음 명령을 실행합니다.

이 명령은 `app/drizzle` 디렉터리에 마이그레이션 파일을 생성합니다.

```bash
pnpm drizzle-kit generate:sqlite
```

`wrangler.toml` 파일에 다음 코드를 추가하여 마이그레이션 파일의 위치를 제공합니다.

```toml
[[d1_databases]]
binding = "DB" # i.e. available in your Worker on env.DB
database_name = "<DB_NAME>"
database_id = "<DB_ID>"
migrations_dir = "./app/drizzle" // [!code ++]
```

다음 명령으로 생성된 마이그레이션 목록을 확인할 수 있습니다.

```bash
pnpm wrangler d1 migrations list <DB_NAME>
```

마이그레이션을 확인했다면, 로컬 데이터베이스에 이를 적용하기 위해 다음 명령을 사용합니다.

이 로컬 데이터베이스는 개발 모드에서 사용됩니다.

```bash
pnpm wrangler d1 migrations apply <DB_NAME> --local
```

로컬 데이터베이스와 프로젝트가 연결되었는지 확인하기 위해 다음과 같은 상호작용을 추가해볼 수 있습니다.

```tsx
// Rmmix loader 함수의 예시

export const loader = async ({ context }: LoaderFunctionArgs) => {
  const db = drizzle(context.cloudflare.env.DB);

  await db.insert(users).values({
    id: 1,
    name: "Junseong Park",
    email: "x@jspark.dev",
    username: "jsparkdev",
  });

  await db.insert(items).values({
    id: 1,
    name: "Macbook Air M1",
    price: 500000,
    description: "My Computer",
    userId: 1,
  });

  const userList = await db.select().from(users);
  const itemList = await db.select().from(items);

  return json({ userList, itemList });
};
```

프로덕션 환경의 데이터베이스에 마이그레이션을 적용하기 위해 다음 명령을 사용합니다.

```bash
pnpm wrangler d1 migrations apply <DB_NAME>
```
