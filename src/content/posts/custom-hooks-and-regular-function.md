사실 커스텀 훅으로 수행할 수 있는 모든 작업은 일반 함수로 수행할 수 있음.
대신, 커스텀 훅은 다른 훅을 사용할 수 있음.
이 말은 상태나 부작용, 컨텍스트 등을 관리하는 로직을 재사용할 수 있다는거임.
상태, 부작용, 컨텍스트에 관한 복잡한 논리를 추상화하여 여러 컴포넌트에서 재사용하려는 목적.

일반 함수는 후크를 사용할 수 없어 상태, 부작용, 컨텍스트 로직을 관리할 수 없음.
대신 리액트와 관련없는 계산, 데이터 처리 등의 작업을 수행하는데 적합함.
앱 전체에서 상태나 라이프사이클과 관련없는 작업을 수행하기 위해 사용.

사실 커스텀 훅도 함수의 일종이라 일반 함수에서 수행할 수 잇는건 다 할수있음. 대신 다른 훅을 호출할 수 있는게 유일한 차이.
