title: Tailwind CSS가 import된 문자열을 인식하지 못하는 문제
description: Tailwind CSS가 import된 문자열을 인식하지 못하는 문제를 해결하는 방법에 대해 알아봅니다.
updatedAt: 2024-05-29
---

Tailwind CSS를 사용하다보면 중복되는 클래스 재사용을 위해 문자열 변수를 특정 파일에 저장한 후 import하여 사용하는게 좋은 경우가 많습니다.

하지만 다음의 경우 Tailwind CSS가 import된 문자열을 인식하지 못하는 문제가 발생할 수 있습니다.

```ts
// /utils/class.ts

export const likeButton = "hover:underline text-white py-2 px-4 rounded-xl";
```
```tsx
// App.tsx
import { likeButton } from "~/utils/class";

<Link to=".." className={`bg-red-500 ${likeButton}`}>
  Cancel
</Link>
```

브라우저의 개발자 도구를 확인해보면 Link 컴포넌트에 문자열은 적용되지만 Tailwind CSS 클래스는 적용되지 않는 것을 확인할 수 있습니다.

이는 Tailwind CSS가 가져온 문자열을 인식하지 못하는 문제로, 구성 파일인 `tailwind.config.ts`에서 해당 파일의 경로를 `content` 배열에 추가하여 문제를 해결할 수 있습니다.

```ts
// tailwind.config.ts
import type { Config } from "tailwindcss";

export default {
  content: [
    "./app/**/*.tsx", 
    "./app/utils/class.ts" // [!code ++]
  ],
  theme: {
    extend: {},
  },
  plugins: [],
} satisfies Config;
```
