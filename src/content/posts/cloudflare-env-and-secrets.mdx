## 환경 변수

환경 변수를 사용하면 문자열 또는 JSON을 Workers에 연결하여 fetch 이벤트 핸들러의 `env` 매개변수를 통해 사용할 수 있습니다. 이 값들은 암호화되지 않으므로 애플리케이션의 구성을 저장하는데 사용됩니다.

환경 변수는 `wrangler.toml` 파일의 `[vars]`에 정의됩니다.

```toml
[vars]
MY_NAME = "Junseong Park"
MY_LAPTOP = { NAME = "Macbook Air M1", YEAR = 2020, COLOR = "Space Gray" }
```

환경 변수의 타입을 정의하기 위해 다음 명령을 실행하여 `worker-configuration.d.ts` 파일에 타입을 추가합니다.

```bash
pnpm wrangler types
```

이제 fetch 이벤트 핸들러의 `env` 매개변수를 통해 환경 변수를 사용할 수 있게 됩니다.

```ts
export default {
	// request 매개변수를 제거하면 오류가 발생합니다.
	async fetch(request: Request, env: Env): Promise<Response> {
		const {
			MY_NAME,
			MY_LAPTOP: { NAME, YEAR, COLOR },
		} = env;

		return new Response(
			`My name is ${MY_NAME} and I use a ${YEAR} ${COLOR} ${NAME}!`,
		);
	},
};
```

개발 서버를 실행한 후, 해당 페이지에 접속하면 다음과 같은 응답을 받을 수 있습니다.

```
My name is Junseong Park and I use a 2020 Space Gray Macbook Air M1!
```

## 비밀

환경 변수는 암호화되지 않으므로 민감한 정보를 저장할 때는 비밀을 사용해야 합니다. 비밀은 암호화되어 저장되는 환경 변수입니다.

개발자는 Wrangler 또는 Cloudflare Dashboard에서 비밀의 값을 확인할 수 없지만, Workers는 비밀을 일반 환경 변수와 똑같이 처리합니다.

비밀은 `wrangler.toml` 파일에 정의하지 않고 프로젝트 루트의 `.dev.vars` 파일에 정의합니다. 

```env
MY_SECRET = This is my secret
```

비밀을 추가한 후, 타입을 생성하면 fetch 핸들러에서 비밀을 사용할 수 있게 됩니다.

이미 배포된 Workers에 비밀을 추가할 수도 있습니다. 이는 두 가지 방법을 통해 가능합니다.

첫 번째 방법은 Cloudflare Dashboard에서 비밀을 직접 추가하는 것입니다. 환경 변수를 추가할 때 암호화 옵션을 활성화하면 데이터를 비밀로 추가할 수 있습니다.

두 번째 방법은 Wrangler를 사용하는 것입니다. 다음 명령을 실행하여 비밀을 추가합니다. 아래 예시에서 `MY_SECRET`는 비밀의 이름입니다.

```bash
wrangler secret put MY_SECRET
```

비밀을 추가하는 방법과 마찬가지로 Cloudflare Dashboard에서 비밀을 제거할 수 있고, Wrangler를 사용하여 비밀을 제거할 수도 있습니다.

```bash
wrangler secret delete MY_SECRET
```
