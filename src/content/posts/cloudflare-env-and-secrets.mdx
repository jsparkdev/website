## 환경 변수

환경 변수를 사용하면 문자열 또는 JSON을 Workers fetch 핸들러의 `env` 매개변수에 추가합니다. 이 값들은 암호화되지 않으므로 주로 구성을 저장하는데 사용됩니다.

### 환경 변수 정의

환경 변수는 `wrangler.toml` 파일의 `[vars]`에 정의됩니다.

```toml
[vars]
MY_NAME = "Junseong Park"
MY_LAPTOP = { NAME = "Macbook Air M1", YEAR = 2020, COLOR = "Space Gray" }
```

환경 변수를 정의한 후, 환경 변수의 자동 완성을 제공하기 위해 다음 명령을 통해 타입을 정의합니다.

```bash
pnpm wrangler types
```

### 환경 변수 사용

Workers fetch 핸들러의 `env` 매개변수를 통해 환경 변수를 사용할 수 있습니다.

```ts
export default {
	// request 매개변수를 제거하면 오류가 발생합니다.
	async fetch(request: Request, env: Env): Promise<Response> {
		const {
			MY_NAME,
			MY_LAPTOP: { NAME, YEAR, COLOR },
		} = env;

		return new Response(
			`My name is ${MY_NAME} and I use a ${YEAR} ${COLOR} ${NAME}!`,
		);
	},
};
```

개발 서버를 실행한 후, 해당 페이지에 접속하면 다음과 같은 응답을 받을 수 있습니다.

```
My name is Junseong Park and I use a 2020 Space Gray Macbook Air M1!
```

## 비밀

비밀은 암호화된 환경 변수이므로 민감한 정보를 저장하는데 사용할 수 있습니다.

### 비밀 정의

환경 변수와 달리 `wrangler.toml` 파일에 정의하지 않고 프로젝트 루트 `.dev.vars` 파일에 정의합니다. 

```
MY_SECRET = This is my secret
```

비밀을 추가한 후, 환경 변수와 마찬가지로 다음 명령을 실행하여 타입을 정의합니다.

```bash
pnpm wrangler types
```

### 배포된 Workers에 비밀 추가

이미 배포된 Workers에 비밀을 추가하는 두 가지 방법이 존재합니다. 

1. Cloudflare Dashboard에서 환경 변수 추가 시 암호화 활성화
2. `wrangler` 사용

```bash
pnpm wrangler secret put MY_SECRET
```
