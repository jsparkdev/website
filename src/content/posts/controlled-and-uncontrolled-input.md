`React`에서 `input`을 다루는 두 가지 방법이 존재합니다. 바로 `Uncontrolled Input`과 `Controlled Input`입니다. 이 두 방법은 각각의 장단점이 존재하며, 상황에 따라 적절한 방법을 선택해야 합니다.

## Uncontrolled Input

우리가 지금까지 한건 언컨트롤드 폼이었음.

인풋의 값을 상태로 관리하지 않습니다. DOM에서 input을 처리하므로 기존 HTML 인풋 관리 방식과 유사.
그러므로 값에 접근하기 위해서는 useRef 훅을 사용합니다.

언컨트롤드는 리액트가 값을 추적하지 않음. 대신 값이 필요하면 ref로 접근해서 가져와야지  
그냥 인풋 값을 필요할때 가져오는 기존 htlm방식

빠르고 간단함. 특히 상태로 관리되지 않기 대문에 값이 업데이트되어도 리렌더링을 일으키지 않음. 그래서 성능이 더 좋음. 하지만 성능 차이는 무시할수잇는정도.
입력값을 실시간으로 추적하는 경우 (실시간 유효성 검사 등)가 아니라면 사용하면 좋음.

즉 DOM에 직접 엑세스해서 제어해야하는 경우, 성능을 조금이라도 개선해야하는ㄴ 경우.

근데 사실상 비제어 인풋의 장점은 단순함과 약간의 성능적 이점 뿐.

## Controlled Input

프로그래밍 방식으로 제어하려는 입력이 있을수도있음. 버튼클릭시 값 설정하거나 사용자가 입력한 값 변경 등임.  
이걸하려면 컨트롤드 폼이 필요함.

즉, 컨트롤드는 input에 value를 그냥 할당하는거임.  
이거장점? 리액트가 특정 표현식의 값과 input의 value가 절대 다를수없음을 보장함.  
왜냐면 표현식의 값을 input.vlaue로 때려박았기때문.

보통 입력값을 state에 저장하면 onChange 핸들러가 해당 값을 최신으로 유지함.  
그냥 잘못된 입력에 대해 오류를 표시하는 대신 잘못된 입력을 허용하지 않으면 더 멋짐  
즉, 오류 state를 제거하고 사용자이름 상태를 관리할거임. 입력되기전에 소문자로 바꿀거임. 그 값을 input에 전달하기 때문에 잘못된값 넣는게 불가능해짐.

즉, 컨트롤드는 말 그대로 리액트가 인풋의 값을 상태로 저장해서 제어함.  
인풋 값이 바뀔때마다 onChange로 상태값도 바뀌므로 실시간으로 리액트 상호작용 가능
