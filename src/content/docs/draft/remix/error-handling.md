Eror handling

에러바운더리에서 useRouteError로 에러 정보를 얻어서 에러를 처리할 수 있음.
근데 이건 여전히 단점이 존재
1. 렌더링 과정의 에러만 처리하며, 렌더링 이후 에러는 처리하지 못함 (사이드이펙트에서 에러발생시켜보셈.)
2. 에러가 타입 안정적이지 않음. 그래서 UI에 에러 관련 데이터를 사용할 수 없음.

타입 안정 문제를 해결하기 위해 리믹스에선 주로 에러 객체보단 응답 객체를 던짐.
이 응답 객체는 status 및 여러 정보를 포함하기 때문.

응답을 던지면 리믹스는 페이지 컴포 대신 에러바운더리를 렌더링함.
여기서 useRouteError는 던져진 응답을 반환.
isRouteErrorResponse를 사용하여 한번 걸러내면 status 객체를 가진 객체만 남음

그러면 이 status에 따라 다른 UI를 렌더링하면 됨.
즉, status와 함께 오류 정보를 제공하기 위해 응답을 던지며, 그 응답은 에러 바운더리의 useRouteERror에 의해 반환되고 isRouteErrorResponse(error)로 status를 가진애만 남길수있음.

에러바운더리 커스텀 컴포넌트로 만들어서 써도 좋음.
어차피 로직이 반복될거기때문.
